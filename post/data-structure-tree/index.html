数据结构：树和二叉树
2020-10-31 12:42   Categories: Tech  

补课

二叉数的性质和存储结构

性质

  1. 在二叉树的第 i 层上至多有 2的i-1次幂 个结点，最少有 1 结点
  2. 深度为 k 的二叉树至多有 2的k次幂 -1 个结点，最少有 k 个结点
  3. 对任何一棵二叉树T，如果其叶子数为 a , 度为 2 的结点数位 b，则 a = b + 1
  4. 具有n个结点的完全二叉树的深度为 log以2为底n的对数 + 1
  5. 懒了，没看

两种特殊形式的二叉树

满二叉树

一棵 深度为 k 的二叉树且有 2的k次幂 -1 个结点的二叉树称为满二叉树

特点

  1. 每层都满
  2. 叶子节点全在底层

完全二叉树

深度为k的具有n个节点的二叉树，当且仅当其每一个节点都与深度为k的满二叉树中编号为1~n的节点一一对应时，称之为完全二叉树

（可以缺少某个叶子节点）

特点

  1. 叶子只能分布在层次最大的两层上
  2. 对任一结点，如果其右子树的最大层次为 i，则其左子树的最大层次必为 i 或 i+1

关系

  • 满二叉树 一定是 完全二叉树
  • 完全二叉树 不一定是 满二叉树

二叉树的存储结构

顺序存储

#define MAXTSIZE 100
#define TElemType char

typedef TElemType SqBiTree[MAXTSIZE];

  • 实现：按满二叉树的结点层次编号，依次存放二叉树中的数据元素

  • 缺点：

    • 大小固定
    • 浪费空间

  • 特点：

    • 结点间关系蕴含在其存储位置中

链式存储

二叉链表

每个结点有两个指针指向其左右孩子

//链式存储结构
//二叉链表
typedef struct BiNode {
    TElemType data;
    struct BiNode *lchild,*rchild;//左右孩子指针
}BiNode,*BiTree;

在n个结点的二叉链表中，有n+1个空指针域

三叉链表

//三叉链表
typedef struct TriTNode{
    TElemType data;
    struct TriTNode *lchild,*parent,*rchild;//左孩子，双亲，右孩子
}TriTNode,*TriTree;

遍历二叉树和线索二叉树

遍历二叉树

顺着某一个条路径，使每个结点均被访问且仅被访问一次

  • 先序遍历，根，左，右
  • 中序遍历，左，根，右
  • 后序遍历，左，右，根

根据遍历确定二叉树 由先序遍历和中序遍历，或由后序遍历和中序遍历，可以确定唯一一棵二叉树

递归算法

/**
 * 先序遍历
 * 若为空，则空操作
 * 若为非空，根，左，右
 */
void PreOrderTraverse(BiTree T) {
    if (T == NULL) return;
    else {
        visit(T);//访问根结点
        PreOrderTraverse(T->lchild);//递归遍历左子树
        PreOrderTraverse(T->rchild);//递归遍历右子树
    }
}

/**
 * 中序遍历
 * 若为空，则空操作
 * 若为非空，左，根，右
 */
void InOrderTraverse(BiTree T) {
    if (T == NULL) return;
    else {
        InOrderTraverse(T->lchild);
        visit(T);
        InOrderTraverse(T->rchild);
    }
}

/**
 * 后序遍历
 * 若为空，则空操作
 * 若为非空，左，右，根
 */
void PostOrderTraverse(BiTree T) {
    if (T == NULL) return;
    else {
        PostOrderTraverse(T->lchild);
        PostOrderTraverse(T->rchild);
        visit(T);
    }
}
