浅谈面向对象三大特性

读完了《Java核心技术卷Ⅰ》第一遍之后，总感觉意犹未尽，感觉知识联系不到一起，因此写此文，主要记录记录我对Java中面向对象的三大特性的理解，见笑。

封装

之前我对封装最直观的理解就类似这种：

public class Animal { 
    private String name;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }
}

即，程序只能通过对象的方法与对象数据进行交互，而不可以直接访问，这可理解为封装。

封装的主要思想是为了隐藏自己内部的实现，不是说内部实现不重要，而是更重要的是对外提供的方法，其实封装的概念很广，不只局限于这种setter，getter方法，有一些方法比如Math.random()，也属于封装，封装其实无处不在。

继承

继承的基本思想是，可以基于已有的类去创建新的类。继承了已有类之后，新的类将会复用被扩展类的全部属性和方法，而且可以增加新的方法和字段。
